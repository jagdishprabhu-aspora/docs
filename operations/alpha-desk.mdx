## Overview

Use Alpha Desk as your primary tool for reviewing customer orders and transactions. You'll learn how to log in securely, search for customer information efficiently, and navigate the available dashboards to resolve customer inquiries.

## What You Can Do in Alpha Desk

Use Alpha Desk to perform these essential functions:

- **Search customer profiles** using ID, phone, or email
- **Track order status** across all stages of processing
- **View transaction details** including payment and beneficiary information
- **Verify beneficiary information** before processing
- **Monitor workflows** for UK and UAE operations
- **Access payment history** to resolve customer inquiries

<Info>
  Alpha Desk is only accessible through VPN. Always connect to Pritunl before attempting to access the system.
</Info>

## How to Access Alpha Desk

Follow these steps to log in securely:

<Steps>
  <Step title="Connect to VPN">
    Launch Pritunl VPN client and establish a secure connection.
    
    <Warning>
      You must be connected to VPN before accessing Alpha Desk. Verify your connection is active before proceeding.
    </Warning>
    
    <Check>
    Look for the Pritunl icon in your system tray showing an active connection.
    </Check>
  </Step>
  
  <Step title="Navigate to Alpha Desk">
    Open your browser and go to the Alpha Desk login page.
  </Step>
  
  <Step title="Enter your credentials">
    Log in using your assigned credentials:
    - **Phone number** (provided during onboarding)
    - **Password** (set during first login)
    
    <Tip>
    Keep your credentials secure and never share them with others. Contact your team lead if you've forgotten your password.
    </Tip>
  </Step>
  
  <Step title="Access the dashboard">
    Click login to access your Alpha Desk dashboard and begin working with customer orders.
  </Step>
</Steps>

## Best Practices

<AccordionGroup>
  <Accordion icon="check" title="Before You Start">
    - Always connect to VPN first
    - Verify your credentials are active
    - Ensure you have necessary permissions
    - Have customer information ready before searching
  </Accordion>
  
  <Accordion icon="search" title="Effective Searching">
    - Use the most specific identifier available (Order ID > User ID > Email)

  </Accordion>
  
  <Accordion icon="shield" title="Security">

  <Accordion icon="clock" title="Efficiency Tips">
    - Keep Order IDs and User IDs handy
    - Use browser bookmarks for quick access
    - Learn keyboard shortcuts where available
    - Familiarize yourself with dashboard layouts
  </Accordion>
</AccordionGroup>

## Common Search Scenarios

### Scenario 1: Customer Inquiry About Order

<Steps>
  <Step title="Get Order ID">
    Ask customer for their Order ID or reference number
  </Step>
  
  <Step title="Search App Server Orders">
    Enter Order ID in App Server Orders search
  </Step>
  
  <Step title="Review Details">
    Check order status, payment status, and any issues
  </Step>
  
  <Step title="Provide Update">
    Inform customer based on findings
  </Step>
</Steps>

### Scenario 2: Payment Verification

<Steps>
  <Step title="Identify Region">
    Determine if UK (LULU) or UAE (Goms/Falcon)
  </Step>
  
  <Step title="Use Appropriate System">
    - UK: Search in LULU using LR or LULU ID
    - UAE: Use Falcon dashboard
  </Step>
  
  <Step title="Verify Payment Status">
    Check payment received and processing status
  </Step>
  
  <Step title="Take Action">
    Process according to payment status
  </Step>
</Steps>

### Scenario 3: Customer Profile Review

<Steps>
  <Step title="Collect Identifier">
    Get User ID, phone, or email from customer
  </Step>
  
  <Step title="User Search">
    Search in User Search section
  </Step>
  
  <Step title="Review Profile">
    Check KYC status, order history, account details
  </Step>
  
  <Step title="Address Query">
    Resolve customer's issue based on profile information
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion icon="triangle-exclamation" title="Cannot Access Alpha Desk">
    **Possible causes:**
    - VPN not connected
    - Incorrect credentials
    - Access permissions not granted
    
    **Solution:**
    1. Verify VPN connection via Pritunl
    2. Confirm credentials with team lead
    3. Request access if permissions missing
  </Accordion>
  
  <Accordion icon="triangle-exclamation" title="Search Returns No Results">
    **Possible causes:**
    - Incorrect ID or typo
    - Wrong search method used
    - Order not yet in system
    
    **Solution:**
    1. Verify the search term with customer
    2. Try alternative search methods
    3. Check if order was recently created
  </Accordion>
  
  <Accordion icon="triangle-exclamation" title="Dashboard Not Loading">
    **Possible causes:**
    - Network connectivity issues
    - VPN connection dropped
    - System maintenance
    
    **Solution:**
    1. Check VPN connection
    2. Refresh the page
    3. Check for system status updates
  </Accordion>
</AccordionGroup>
